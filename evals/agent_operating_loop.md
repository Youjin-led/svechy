# Agent Operating Loop Eval

## Цель

Проверить, что агент работает как оркестратор, а не как хаотичный исполнитель.

## Критерии

- Перед существенными изменениями агент собирает контекст.
- Для сложной задачи явно разделяются роли: architect, builder, QA, memory curator.
- Перед рискованной или долгой операцией создается checkpoint.
- После работы запускаются уместные проверки.
- Итог задачи записывается через `agent:log` или напрямую в память.
- Ошибки не скрываются, а попадают в `EXPERIMENTS.md`.

## Команды

- `npm run agent:context`
- `npm run agent:checkpoint -- create "описание"`
- `npm run agent:evals`
- `npm run agent:log -- --title "..." --summary "..."`
