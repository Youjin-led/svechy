# Evals

Эта папка хранит проверочные сценарии для агента. Они нужны, чтобы оценивать качество изменений не по ощущениям, а по явным критериям.

## Базовые критерии

- Агент изучает существующую структуру перед изменениями.
- Изменения остаются в рамках задачи.
- `node_modules/` не редактируется.
- После frontend/3D-изменений запускается `npm run qa`.
- Важные решения попадают в `DECISIONS.md`.
- Неудачные или спорные попытки попадают в `EXPERIMENTS.md`.
- Долгие задачи получают запись в `TASKS.md` или `PLAN.md`.
